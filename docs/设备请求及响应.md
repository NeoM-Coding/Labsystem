# 设备请求及响应
文档用于描述各种设备支持的指令以及指令对应的响应，指导如何使用node.js mock下位机的响应数据
务必使用ts构建

请求和响应 采用json描述，实际发送和回复的应该是payload中的byte数组
```json lines
{
  "commandLine": "",
  "request": {
    "payload": [],
    "address_index": 0,
    "self_id_index": 1 // 可选字段 取决于设备是否支持
  },
  "response": {
    "payload": [],
    "address_index": 0,
    "self_id_index": 1 // 可选字段 取决于设备是否支持
  }
}
```
其中commandLine参数来自
```java
/**
 * {@link xyz.jasenon.lab.common.command.CommandLine}
 */
```
response则参考我给出的实例，根据请求的payload替换对应的address, self_id(若有)

设备通过地址范围区分类型

| 地址范围 | 设备类型 |
|------|------|
| 1-10 | Access |
| 11-30 | CircuitBreak |
| 31-40 | AirCondition |
| 41-60 | Light |
| 61-80 | Sensor |

mock端也要处理校验位，处理方式参考
```java
/**
 * {@link xyz.jasenon.lab.mqtt.client.mqtt.MqttTask}
 * 其中的内部类Explainer
 */
```
具体实现位于
```java
/**
 * {@link xyz.jasenon.lab.common.command.checker.CrcChecker} CRC校验
 * {@link xyz.jasenon.lab.common.command.checker.SumChecker} 有符号和无符号校验 核心就是FF的处理  是当作0 还是-1
 */
```

## Access 请求

-

1. 单次开门请求

```json lines
// 若access.address = 1

  "request": {

  },
  "response": {
    "payload": [1,10,2,255,13],
```
